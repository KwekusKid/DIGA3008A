Reflection on final website

Reflective Essay: Building an Editorial-Inspired Minimalist Web Architecture

The design and development of my website for this website was guided by a blend of technical rigor and visual minimalism, inspired by avant-garde fashion websites like Martine Rose and Wales Bonner, as well as editorial platforms such as The New Yorker, MUBI, and Frank Ocean’s Homer. These influences shaped both the visual aesthetic, clean, minimal, spacious, and the underlying structure, which prioritizes clarity, modularity, and accessibility. I want my website to be a balance between an experience and a functional website that it’s to navigate and one that exemplifies good practice. Majority of my UI and UX decisions were made as result of this goal.

Folder Structure

The folder structure of my project reflects a clear hierarchical information architecture. Root-level folders such as CSS, javascript, Images, Portfolio, and Profile are designed to separate concerns and promote modular scalability. HTML files (like index.html) sit at the root, representing the structural content layer. CSS files (e.g., index.css, normalize.css) handle the presentation layer, while JavaScript logic is isolated in its own directory. This modularity supports both long-term maintainability and team-based scaling, aligning with industry-standard best practices.

Semantic Markup

In crafting the HTML, I prioritized semantic structure to enhance both usability and accessibility. Elements like header, nav, main, and section provide clear landmarks for screen readers and improve browser interpretation of the content. Interface elements, such as navigation links and content blocks, are expressed using appropriate semantic tags. For example, project showcases within the portfolio section are structured with article and figure where appropriate, communicating meaning beyond mere layout. Semantic markup here improves keyboard navigation, search engine indexing, and accessibility, particularly for users relying on assistive technologies.

Metadata

Each page includes descriptive metadata, including meta charset="UTF-8" and *meta name="viewport" for responsive scaling. Metadata such as page descriptions, keywords, and titles guide browser rendering and inform SEO, ensuring that the content is discoverable and correctly displayed across devices. As the site evolves, metadata will also help structure new content types within the Blogs or Essays directories.

CSS

All styling is externalized via modular CSS files. This ensures consistency across pages and avoids in-line styling, which would compromise maintainability. The design employs a grid-based layout, subtle typography, and editorial spacing that evoke the clean aesthetic of avant-garde design. Responsiveness is achieved through @media queries, ensuring the site adapts to various screen sizes, from desktop to mobile, while preserving its minimal aesthetic. The use of normalize.css further ensures baseline consistency across browsers.

JavaScript

While interactivity is minimal, consistent with the site's editorial tone, any interaction points (e.g., toggling menus, content filtering) are handled via external JS files. DOM manipulation and event handling are used to keep the logic layer separate and maintain a clean HTML structure. No in-line scripts are used, and asynchronous loading techniques are planned for future media-heavy additions to enhance performance and user experience.

Conclusion

The final product reflects a careful negotiation between form and function. The minimalist visual design, borrowed from avant-garde and editorial aesthetics, works in tandem with a robust modular architecture that separates content, presentation, and logic. Semantic markup, metadata, and accessibility considerations ensure the site isn’t just visually appealing but also usable, adaptable, and inclusive. This site isn’t just a personal portfolio—it’s a reflection of how digital spaces can be both intentional and expansive, structured and expressive.
